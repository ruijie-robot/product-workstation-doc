### 3.3 数据流

  

**基金筛选流程：**

```

用户输入自然语言 → Supabase Edge Function

→ LLM 解析意图 (Function Calling: screen_funds/screen_managers)

→ TypeScript 执行筛选逻辑（遍历 Mock 基金数据库，逐条件过滤、排序）

→ 返回结构化结果 (ScreenerPayload)

→ 前端渲染 ScreenerPanel 表格

```

  

**基金绩效分析流程：**

```

用户输入自然语言 → Supabase Edge Function

→ LLM 解析意图 (Function Calling: generate_fund_deep_analysis)

→ TypeScript: 查找基金 (findFundFuzzy)

→ TypeScript: 选取同类对比基金 (pickSimilarFunds)

→ TypeScript: 模拟净值曲线 (buildNavSeries)

→ LLM: 调用 Gemini 生成定性分析（概述、亮点、不足、事件评价、结论）

→ 组合返回 DeepAnalysisPayload

→ 前端渲染 DeepAnalysisPanel 报告

```

### 5.2 主题样式

  

- **主题色**：橙色（Orange），shadcn/ui 默认主题变体

- **A 股配色**：红色代表上涨，绿色代表下跌（PctCell 组件自动处理）

- **暗色模式**：支持暗色模式切换

  

### 5.3 主要面板说明

  

#### 筛选结果面板（ScreenerPanel）

  

- **头部**：展示"AI 筛选"徽标、筛选模式（基金筛选/基金经理筛选）、匹配总数、原始查询语句

- **筛选标签**：已应用的筛选条件以 Badge 形式展示

- **基金结果表格**：9 列（代码、简称、公司、类别、经理、近1年年化、近1年最大回撤、夏普、规模）

- **经理结果表格**：8 列（经理、公司、主投策略、在管规模、任职年限、年化回报、最大回撤、代表产品）

- **交互**：点击基金行触发深度分析

- **免责声明**：底部提示"数据库覆盖有限，仅供演示。结果由 AI 根据自然语言筛选条件生成，仅供研究参考，不构成投资建议。"

  

#### 深度分析面板（DeepAnalysisPanel）

  

- **概览区**：对象名称、代码、公司/类别/经理标签、4 大指标卡

- **净值走势图区**：LineChart（Recharts），包含目标基金曲线 + 可选基准/对比基金线，6 大市场事件以 ReferenceArea 色块叠加（牛市绿/熊市红/震荡琥珀/政策蓝/中性灰）

- **事件表现拆解**：每个事件区间的收益、超额收益、AI 点评卡片，颜色编码正/负/中性

- **亮点/不足**：双栏对比卡片，CheckCircle2/XCircle 图标

- **同类对比**：更优/更弱对比卡片 + 叠加对比净值曲线

- **综合结论**：Markdown 渲染的总结性评价

- **免责声明**：底部提示"本分析由 AI 即时生成，净值与事件区间为模拟构造，仅供研究参考，不构成投资建议。"

## 六、AI 函数调用清单

  

AI 助手一期支持的 Function Calling 能力：

  

| 函数名 | 功能 | 触发条件 |

|--------|------|---------|

| `screen_funds` | 按自然语言条件筛选基金 | 用户要求按条件筛选基金（"筛选近1年收益前10的偏股混合基金"） |

| `screen_managers` | 按自然语言条件筛选基金经理 | 用户要求按条件筛选经理（"固收+年化收益4.5%以上的经理"） |

| `generate_fund_deep_analysis` | 生成基金深度分析报告 | 用户要求深度分析某只基金（"分析广发稳健增长A"） |

| `generate_manager_deep_analysis` | 生成基金经理深度分析报告 | 用户要求深度分析某位经理（"分析刘格菘"） |

  

### 筛选函数参数详解

  

#### `screen_funds`

  

| 参数 | 类型 | 说明 |

|------|------|------|

| `query` | string（必填） | 用户原始自然语言查询 |

| `company` | string | 基金公司名称（模糊匹配） |

| `category` | string | 基金类别（偏股混合/固收+/纯债/股票型/QDII/量化增强/ETF） |

| `annualReturnMin` | number | 最低年化收益率（%） |

| `annualReturnMax` | number | 最高年化收益率（%） |

| `maxDrawdownMax` | number | 最大回撤上限（%） |

| `sharpeMin` | number | 最低夏普比率 |

| `volatilityMax` | number | 最大年化波动率（%） |

| `upDownCaptureMin` | number | 最低涨跌比 |

| `aumRankTop` | number | 按规模取前 N 名 |

| `sortBy` | string | 排序方式（"return"/"drawdown"/"aum"/"sharpe"） |

| `limit` | number | 返回结果数量上限（默认 20） |

  

#### `screen_managers`

  

| 参数 | 类型 | 说明 |

|------|------|------|

| `query` | string（必填） | 用户原始自然语言查询 |

| `company` | string | 所在公司 |

| `strategy` | string | 主投策略 |

| `tenureYearsMin` | number | 最低任职年限 |

| `avgAnnualReturnMin` | number | 最低年化回报（%） |

| `maxDrawdownMax` | number | 最大回撤上限（%） |

| `totalAumMin` | number | 最低在管规模（亿） |

| `sortBy` | string | 排序方式（"return"/"drawdown"/"aum"/"tenure"） |

| `limit` | number | 返回结果数量上限（默认 20） |

  

#### `generate_fund_deep_analysis`

  

| 参数 | 类型 | 说明 |

|------|------|------|

| `subject` | string（必填） | 基金名称或代码 |

  

#### `generate_manager_deep_analysis`

  

| 参数 | 类型 | 说明 |

|------|------|------|

| `subject` | string（必填） | 基金经理姓名 |

## 七、开发与部署

  

### 7.1 开发环境配置

  

**环境要求：**

- Node.js >= 18

- npm 或 bun

  

**环境变量（`.env`）：**

```

VITE_SUPABASE_URL=<supabase-project-url>

VITE_SUPABASE_PUBLISHABLE_KEY=<supabase-anon-key>

```

  

**启动开发服务器：**

```bash

npm install

npm run dev

```

开发服务器默认运行在 `http://localhost:8080`。

  

### 7.2 构建与部署

  

```bash

npm run build # 生产构建

npm run preview # 预览构建产物

```

  

构建产物输出到 `dist/` 目录，可部署到任何静态托管服务（Vercel、Netlify、Cloudflare Pages 等）。

  

Supabase Edge Function 需单独部署：

```bash

supabase functions deploy chat

```
